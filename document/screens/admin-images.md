# 画像管理ページ

## 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/admin/images` |
| ファイル | `src/pages/admin/images.astro` |
| レンダリング | SSR（サーバーサイドレンダリング） |
| 認証 | Cloudflare Accessで保護 |

## 概要

Cloudflare R2への画像アップロード・一覧表示・URLコピー・削除を行う管理画面。ドラッグ＆ドロップによる複数ファイルの同時アップロードに対応。

## アクセス制御

- Cloudflare Accessによる認証が必要
- R2バケット（`BUCKET`）が未設定の場合はトップページにリダイレクト

## 画面構成

### ヘッダー

- サイトタイトル「Bikyu-Diary」（トップページへのリンク）

### アップロードエリア（カード内）

- ページタイトル「画像アップロード」（`<h1>`）
- ドラッグ＆ドロップゾーン（`drop-zone` クラス）
  - アップロードアイコン（SVG）
  - 説明テキスト「ここに画像をドラッグ＆ドロップ」
  - 「または」テキスト
  - 「ファイルを選択」ボタン（`<label>` + 非表示 `<input type="file">`）
  - 複数ファイル選択可能（`multiple` 属性）
  - accept属性: `.jpg,.jpeg,.png,.gif,.webp`
- アップロード中表示
  - 進捗テキスト「アップロード中... (n/N)」
- アップロード結果（成功時に表示）
  - 「アップロード成功」タイトル
  - アップロードされた画像のプレビュー（最大幅300px）
  - 「URLコピー」ボタン
  - 「マークダウンコピー」ボタン
- エラー表示エリア（赤背景、エラー時のみ表示）

### 画像一覧（カード内）

- タイトル「アップロード済み画像」（`<h2>`）
- グリッドレイアウト（`grid-template-columns: repeat(auto-fill, minmax(200px, 1fr))`）
- 最新6件を表示
- 各画像アイテム（`image-item` クラス）
  - サムネイル画像（高さ150px、`object-fit: cover`）
  - 操作ボタン（縦並び）
    - 「URLコピー」ボタン
    - 「マークダウンコピー」ボタン
    - 「削除」ボタン（赤、確認ダイアログ付き）
- 画像0件時: 「まだ画像がありません。」

### R2ダッシュボードリンク

- 「R2 ダッシュボード →」（外部リンク、Cloudflareダッシュボードへ）

### 戻るボタン

- `← 管理画面に戻る`（`/admin` へ遷移）
- スタイル: `btn btn-secondary`
- ラッパー: `<div class="mt-lg">`

### フッター

- コピーライト「© {年} Bikyu-Diary」

## ドラッグ＆ドロップ操作

### ドラッグオーバー時

- ドロップゾーンのボーダーカラーがプライマリカラーに変化
- 背景に薄い青色のハイライト

### ドロップ時

- ドロップされたファイルを順次アップロード

## 操作フロー

### アップロード

1. ドラッグ＆ドロップまたは「ファイルを選択」でファイルを指定
2. クライアント側でファイル形式をチェック（JPEG, PNG, GIF, WebP のみ）
3. ドロップゾーンがアップロード中表示に切り替わり、進捗を表示
4. 各ファイルを `POST /api/upload?index={n}` で順次アップロード
5. 全ファイル完了後、ドロップゾーンを元に戻し、画像一覧を再読み込み
6. 最後にアップロードした画像のプレビューと操作ボタンを表示
7. エラーがある場合は成功件数・失敗件数を表示

### URLコピー

1. 「URLコピー」ボタンをクリック
2. 画像のURL（例: `https://r2diary.bikyu.dev/images/2026032800000001.jpg`）をクリップボードにコピー
3. 「コピーしました」アラートを表示

### マークダウンコピー

1. 「マークダウンコピー」ボタンをクリック
2. `![](https://r2diary.bikyu.dev/images/2026032800000001.jpg)` 形式でクリップボードにコピー
3. 「コピーしました」アラートを表示

### 削除

1. 「削除」ボタンをクリック
2. 確認ダイアログ「この画像を削除しますか？」
3. OKで `DELETE /api/{key}` を呼び出し（keyには `images/` プレフィックスが含まれる）
4. 成功時、画像一覧を再読み込み
5. エラー時、エラーメッセージを表示

## バリデーション

### クライアント側

| チェック | 動作 |
|----------|------|
| 許可されていないファイル形式 | アップロードをスキップ（失敗としてカウント） |

### サーバー側（`POST /api/upload`）

| チェック | エラーメッセージ | ステータス |
|----------|----------------|-----------|
| ファイル未選択 | 「ファイルが選択されていません」 | 400 |
| 許可されていない形式 | 「jpg, png, gif, webp のみアップロードできます」 | 400 |

### 許可するファイル形式

| 形式 | MIMEタイプ |
|------|-----------|
| JPEG | image/jpeg |
| PNG | image/png |
| GIF | image/gif |
| WebP | image/webp |

## API

### 画像アップロード

| 項目 | 内容 |
|------|------|
| エンドポイント | `POST /api/upload` |
| ファイル | `src/pages/api/upload.ts` |
| クエリパラメータ | `index` - ファイル連番（デフォルト: 1） |
| リクエスト | `multipart/form-data`（`file` フィールド） |

**ファイル命名規則:**
- 形式: `images/{yyyymmddHHMMSS}{2桁連番}.{拡張子}`
- タイムゾーン: JST（UTC+9）
- 例: `images/2026032815300001.jpg`

**レスポンス（成功時 / 201）:**
```json
{
  "url": "https://r2diary.bikyu.dev/images/2026032815300001.jpg",
  "key": "images/2026032815300001.jpg"
}
```

**レスポンス（エラー時 / 400, 500）:**
```json
{
  "error": "エラーメッセージ"
}
```

### 画像一覧取得

| 項目 | 内容 |
|------|------|
| エンドポイント | `GET /api/images` |
| ファイル | `src/pages/api/images.ts` |

- R2バケットから `images/` プレフィックスの画像を取得
- アップロード日時の降順（新しい順）でソート
- 最大6件に制限

**レスポンス（成功時 / 200）:**
```json
{
  "images": [
    {
      "key": "images/2026032815300001.jpg",
      "url": "https://r2diary.bikyu.dev/images/2026032815300001.jpg",
      "size": 123456,
      "uploaded": "2025-01-01T00:00:00.000Z"
    }
  ]
}
```

### 画像削除

| 項目 | 内容 |
|------|------|
| エンドポイント | `DELETE /api/images/{key}` |
| ファイル | `src/pages/api/images/[...key].ts` |

- パスパラメータの `key` に `images/` プレフィックスを付与してR2から削除

**レスポンス（成功時 / 200）:**
```json
{
  "success": true
}
```

**レスポンス（エラー時 / 400, 500）:**
```json
{
  "error": "エラーメッセージ"
}
```
