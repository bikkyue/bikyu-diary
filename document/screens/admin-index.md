# 管理画面トップページ

## 基本情報

| 項目 | 内容 |
|------|------|
| URL | `/admin` |
| ファイル | `src/pages/admin/index.astro` |
| レンダリング | SSR（サーバーサイドレンダリング） |
| 認証 | Cloudflare Accessで保護 |

## 概要

管理者向け機能の一覧を表示するダッシュボードページ。各管理機能へのナビゲーションを提供する。

## アクセス制御

- Cloudflare Accessによる認証が必要
- `runtime.env.BUCKET` が未設定の場合はトップページ（`/`）にリダイレクト
- 既存の `admin/images.astro` と同じ認証パターンを踏襲

## 画面構成

### ヘッダー

- サイトタイトル「Diary」（トップページへのリンク）
- 共通レイアウト（`Layout.astro`）を使用

### メインコンテンツ

- ページタイトル「管理画面」（`<h1>`）
- 機能カード一覧（グリッドレイアウト）

### 機能カード

各管理機能を1枚のカードとして表示する。

| 機能名 | リンク先 | 説明 |
|--------|---------|------|
| 画像管理 | `/admin/images` | 画像のアップロード・一覧・削除 |

※今後、機能追加時にカードを増やしていく想定。

#### カードの構成要素

- アイコン（SVG）
- 機能名（`<h2>`）
- 簡単な説明テキスト
- カード全体がリンクとしてクリック可能

### 戻るボタン

- コンテンツ下部に `← 一覧に戻る` ボタンを配置
- リンク先: `/`（トップページ）
- スタイル: `btn btn-secondary`（日記詳細ページの戻るボタンと同一）
- ラッパー: `<div class="mt-lg">`

### フッター

- コピーライト「© {年} Diary」

## スタイル

- カードは `.card` クラスをベースに、ホバー時にボーダーカラーを変化させる
- グリッドレイアウト: `grid-template-columns: repeat(auto-fill, minmax(250px, 1fr))`
- 既存の `global.css` の変数（`--spacing-*`, `--border-color`, `--primary-color` 等）を使用

## 実装方針

1. `src/pages/admin/index.astro` を新規作成
2. 既存の `admin/images.astro` と同じ認証・リダイレクトパターンを使用
3. `Layout.astro` コンポーネントを使用（`title="管理画面"`）
4. 機能カードは将来の拡張を考慮し、配列で管理して `map` で描画

## 関連ファイル

| ファイル | 説明 |
|---------|------|
| `src/pages/admin/index.astro` | 管理画面トップ（新規作成） |
| `src/pages/admin/images.astro` | 画像管理画面（既存・戻るボタン追加） |
| `src/components/Layout.astro` | 共通レイアウト（既存） |
