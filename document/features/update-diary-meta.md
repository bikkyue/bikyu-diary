# 日記メタデータ自動更新ワークフロー

## 概要

`src/content/diaries/` 配下の `.md` ファイルが `main` ブランチに push されたとき、GitHub Actions が `src/content/diaries.json` を自動で更新する。

## ワークフローファイル

`.github/workflows/update-diary-meta.yml`

## トリガー条件

- ブランチ: `main`
- 対象パス: `src/content/diaries/**`（直下・サブディレクトリ含む全ファイル）

## diaries.json の形式

```json
{
  "テスト日記.md": "2026-03-28 00:00:00",
  "2026-03/新しいキーボードを買った.md": "2026-03-15 20:00:00"
}
```

- キー: `src/content/diaries/` からの相対パス（`.md` 拡張子込み）
- 値: 日本時間の日時（`YYYY-MM-DD HH:mm:ss` 形式）

## 処理フロー

### 1. 再帰スキャン

`src/content/diaries/` 配下を再帰的に走査し、全 `.md` ファイルの相対パスを収集する。

### 2. メタデータの洗い替え

収集した各ファイルについて、以下の優先順で `created_at` を決定する。

| 優先度 | 条件 | 動作 |
|--------|------|------|
| 1 | 相対パスが旧JSONにそのまま存在する | 日付をそのまま保持 |
| 2 | パスは異なるが、同じファイル名のエントリが旧JSONに存在する | 旧エントリの日付を引き継ぐ（ディレクトリ移動対応） |
| 3 | いずれにも該当しない | 新規ファイルとして現在時刻（JST）を付与 |

### 3. 孤立エントリの削除

旧JSONに存在するが、現在のファイルシステムに対応するファイルがないキーは削除される。

### 4. コミット・プッシュ

差分がある場合のみ、`github-actions[bot]` として `diaries.json` をコミット・プッシュする。

```
chore: update diary metadata
```

## ファイル移動時の日付保持（洗い替え）

ファイルをディレクトリ間で移動した場合、ファイル名（basename）で突合して作成日を引き継ぐ。

**例:**

```
移動前: テスト日記.md → "2026-03-28 00:00:00"
移動後: 2026-03/テスト日記.md → "2026-03-28 00:00:00"（日付保持）
```

旧キー `テスト日記.md` は自動で削除され、新キー `2026-03/テスト日記.md` に日付が引き継がれる。

## 注意事項

- **ファイル名の一意性**: 異なるディレクトリに同名ファイルが存在する場合、ファイル名による突合は先勝ちとなり、片方に新しいタイムスタンプが付与される可能性がある。
- **手動編集**: `diaries.json` を手動で編集することも可能。ワークフローは既存エントリの日付を上書きしない。
- **差分なし時**: ファイル構成に変更がなければコミットは作成されない。
